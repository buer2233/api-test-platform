# 2026-04-08 移动端引导页原型设计说明

## 1. 任务目标
在当前仓库尚未建立正式 React 前端工程的前提下，补充一个可直接预览的移动端 App 引导页原型，用于承接“可用型入口”设计方向中的移动端首屏体验探索。

## 2. 原型定位
本次原型不属于正式业务实现，属于 UI 设计与页面原型补充，目标是提供以下能力：
1. 直接打开即可预览；
2. 体现插图风格；
3. 体现步骤式引导流程；
4. 可作为后续 React + TypeScript 页面拆分参考。

## 3. 视觉方向
本次原型沿用 `UI设计说明文档.md` 中已确认的方向：
1. 深色中性基调；
2. 平台感与工具感并存；
3. 避免普通后台模板风格；
4. 使用克制但有辨识度的插图块和光感层次。

同时，为了适配移动端引导页场景，做如下局部强化：
1. 用柔和渐变和 SVG 插图建立首次进入的情绪价值；
2. 用三步卡片明确“引导流程”而不是仅展示卖点；
3. 用结果预告区减少用户对引导流程的跳失。

## 4. 页面结构
原型文件路径：
- `docs/ui-prototypes/mobile-onboarding-illustrated.html`

页面区块拆分如下：
1. 顶部说明区：承载原型标题与定位说明；
2. 手机画布外框：模拟真实移动端预览环境；
3. Hero 区：品牌、标题、摘要和主插图；
4. 价值速览区：用三个短指标说明引导收益；
5. 步骤流程区：三张步骤卡片，对应引导流程；
6. 结果预告区：说明完成引导后的直接收益；
7. CTA 区：主操作与次操作；
8. 分页指示器：为后续多页引导扩展预留位置。

## 5. 组件拆分建议
后续如接入正式前端工程，建议拆分为：
1. `MobileFrame`
2. `OnboardingHero`
3. `OnboardingValueStrip`
4. `OnboardingStepCard`
5. `OnboardingSummary`
6. `OnboardingActions`
7. `OnboardingPager`

## 6. 当前边界
本次仅交付静态原型，不包含：
1. 页面路由；
2. 真实交互逻辑；
3. 接口联调；
4. 动态数据加载；
5. Windows 壳适配实现。

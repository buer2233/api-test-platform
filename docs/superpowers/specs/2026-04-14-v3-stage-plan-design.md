# V3 阶段计划设计说明

## 1. 文档说明
- **文档名称**：V3 阶段计划设计说明
- **设计日期**：2026-04-14
- **文档目的**：归档本轮围绕 V3 阶段工作计划文档所完成的多轮需求确认结果，作为正式计划文档的设计依据。

## 2. 设计结论
本轮已确认 V3 的核心设计结论如下：

1. V3 的第一主轴为 **平台治理优先**；
2. V3 采用 **P0 / P1 / P2 分层推进**；
3. `P0` 聚焦 **多项目与环境治理**，并明确做到 **执行隔离级**；
4. `P0` 的入口形态采用 **浏览器端最小治理入口**，用于验证项目切换、环境绑定、场景集筛选、版本查看和隔离效果；
5. `P1` 承接 **权限体系、抓包正式执行闭环、产品入口深化、调度与执行中心**；
6. `P2` 只保留 **高级 AI 治理边界与独立验收要求**，不把自动自愈作为当前轮承诺交付；
7. `macOS` 保留为后续要做，但 **当前阶段暂不做正式实现**；
8. `Windows` 在 V3 中需要产出可实际测试的 Demo，但采用 **先 Web 稳定、再轻量套壳到 Windows** 的路线；
9. Windows 壳方案默认采用 **Tauri 优先，浏览器先验，阶段性打包复验**；
10. `P0 / P1 / P2` 都必须各自形成独立验收，而不是只做总体验收。

## 3. P0 设计范围
P0 已确认包含以下能力：

1. 项目对象；
2. 测试环境对象；
3. 场景集、标签、归档与基线版本；
4. 项目级工作区隔离、执行记录隔离、导入导出边界隔离和环境绑定执行；
5. V2 历史资产迁入默认项目 / 默认环境 / 默认场景集；
6. 浏览器端最小治理入口。

其中已确认：
- 环境管理深度为 **测试环境层**；
- 场景集与版本治理深度为 **基线版本级**；
- 历史数据承接策略为 **默认项目迁移**。

## 4. P1 / P2 设计范围
P1 已确认包含：
1. 权限体系与审计治理；
2. 抓包正式执行闭环；
3. 产品入口深化；
4. 调度与执行中心。

P2 已确认只包含：
1. 多轮建议边界；
2. 审批门禁；
3. 回退要求；
4. 失败回滚原则；
5. AI 治理测试设计与独立验收要求。

## 5. 非目标
本轮已明确不纳入 V3 当前必达范围的内容包括：

1. `macOS` 正式客户端实现；
2. 企业治理级配额与复杂共享市场；
3. 更重的分支 / 合并 / 回滚协作版本治理；
4. 完全自治的 AI 自动修复与无人审批执行；
5. 一次性做成完整企业级测试平台。

## 6. 验收原则
V3 已确认的验收原则为：

1. `P0` 独立验收；
2. `P1` 独立验收；
3. `P2` 独立验收；
4. 最终再进行 V3 总体验收。

其中 `P0` 的关键验收点包括：
1. 历史资产迁移成立；
2. 项目 / 环境 / 场景集 / 版本 / 执行记录统一归属成立；
3. 项目级执行隔离成立；
4. 浏览器端最小治理入口完成真实闭环；
5. 自动化测试、迁移检查和入口冒烟通过。

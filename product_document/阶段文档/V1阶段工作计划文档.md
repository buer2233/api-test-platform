## 2. V1 的定位
V1 不是最终产品成品，而是：
> **需求验证版 / 架构奠基版 / 文档驱动最小闭环版**
V1 的核心价值不是“大而全”，而是：
1. 验证产品方向正确；
2. 验证架构方向可落地；
3. 改造现有框架，使其能承接 AI 生成资产；
4. 打通文档驱动最小闭环；
5. 建立后续 V2 / V3 扩展基础。
---
## 3. V1 总目标
V1 需要完成的核心目标如下：
1. 完成产品、架构、中间模型、UI 设计和框架改造文档沉淀；
2. 明确并建立统一中间模型雏形；
3. 明确并建立模板体系雏形；
4. 明确并建立基础规则体系雏形；
5. 明确现有 `api_test` 框架的改造方向与过渡方案；
6. 明确 Web 与 Windows 客户端统一风格的 UI 设计方向；
7. 打通文档驱动生成接口方法与基础测试骨架的最小闭环；
8. 打通 pytest 执行与基础报告输出的最小闭环；
9. 建立生成记录、执行记录、阶段进度记录的基础机制。
---
## 4. V1 不做什么
为了保证范围收敛，V1 当前明确不要求完成以下内容：
1. 三条输入路线全部完整打通；
2. 抓包驱动完整闭环；
3. 功能测试用例驱动完整场景编排；
4. 完整桌面应用最终成品；
5. 完整 Web 产品化交互功能闭环；
6. 多人协作 / 权限 / 中心化管理；
7. 高级 AI 自愈与自动修复能力；
8. 面向非技术用户的完整低门槛交互体验。
### 4.1 但 V1 必须完成的新增前置项
1. 需要完成独立 UI 设计说明文档；
2. 需要明确 Web 与 Windows 客户端的统一视觉风格；
3. 需要明确后端按 Django + DRF 设计；
4. 需要明确数据存储按 MySQL 设计。

## 5. V1 最小闭环定义
V1 的“最小闭环”定义如下：
### 5.1 输入
用户提供：
- OpenAPI / Swagger / 结构化接口文档（优先）
- 或可被标准化解析的接口说明文档
### 5.2 处理
平台完成：
1. 文档解析
2. 标准化处理
3. 转换为统一中间模型
4. 基于模板生成接口方法资产
5. 基于模板生成基础 pytest 测试骨架
6. 规则校验
7. 记录生成信息
### 5.3 输出
平台产出：
- 接口方法代码
- 测试用例骨架代码
- 基础生成记录
### 5.4 验证
平台能够：
- 调用 pytest 执行最小测试闭环
- 输出执行结果
- 记录执行状态
### 5.5 闭环完成标准
当平台能完成以下事情时，可视为 V1 最小闭环打通：
1. 从文档输入得到结构化中间模型；
2. 从中间模型生成符合规范的接口方法；
3. 从中间模型生成符合规范的测试骨架；
4. 生成结果通过规则校验；
5. 能完成 pytest 执行；
6. 有基础报告和执行记录。

## 7. V1 详细设计方案
## 7.1 文档驱动设计方案
### 设计思路
优先从结构化程度最高、最适合批量生成的“接口文档驱动”入手。
### 设计目标
- 从 OpenAPI / Swagger 中提取接口定义；
- 转换为统一中间模型；
- 再通过模板生成接口代码与测试骨架。
### 设计收益
- 最容易验证整体链路；
- 最适合作为 V1 最小闭环；
- 为 V2 的场景驱动和抓包驱动打基础。
---
## 7.2 中间模型设计方案
### 核心要求
- 所有输入不直接跳到代码；
- 先标准化，再转模型；
- 模型可被模板、规则、执行层共同消费。
### 设计收益
- 三条路线未来都能统一承接；
- 降低自由生成不稳定性；
- 便于记录与追溯。
---
## 7.3 模板设计方案
### 核心要求
- 所有代码生成优先走模板；
- 模板与生成逻辑解耦；
- 模板与规则可独立演进。
### 模板输出对象
- 接口方法
- 测试用例骨架
- 断言片段
- 记录文件
---
## 7.4 规则设计方案
### 核心要求
规则层在 V1 中先做基础规则，不追求复杂治理。
### 基础规则至少包括
- 文件命名规则
- 模块归类规则
- 接口函数命名规则
- 测试文件与测试函数命名规则
- 参数字段完整性规则
---
## 7.5 执行设计方案
### 核心要求
V1 不做复杂调度，先把最小执行闭环跑通。
### 执行阶段建议
1. 结构校验
2. 规则校验
3. pytest 执行
4. 结果归档
---

1. 产品/架构说明书先行
2. 详细测试设计先行
3. TDD

1. 建立文档体系
2. 建立模型定义方向
3. 建立框架改造方向

## 9. V1 测试方案
## 9.1 测试目标
V1 的测试目标不是覆盖最终所有业务，而是验证：
1. 文档驱动路线可通；
2. 模型结构合理；
3. 模板策略可落；
4. 规则校验可工作；
5. pytest 执行闭环可成立。
## 9.2 测试层次
1. 解析层测试
2. 模型层测试
3. 模板层测试
4. 规则层测试
5. 执行层测试
6. 最小闭环集成测试
## 9.3 测试通过标准
V1 阶段所有说明书、设计项和实现项，必须有对应测试与验证结果支持，不能口头宣称完成。
